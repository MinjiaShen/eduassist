## 中医门诊病历图像提取工具（RapidOCR 本地版）

从中医门诊管理系统截图中自动提取患者信息、诊断、处方等数据，并输出为结构化的 Excel 表格。采用 RapidOCR 本地识别引擎，无需 API Key，数据不出本机。

---

### 核心功能

- **OCR 文字识别**：基于 RapidOCR（PaddleOCR 模型 + ONNX Runtime），支持中文医疗文本识别
- **结构化提取**：自动识别患者姓名、性别、年龄、电话、主治医师、科室、就诊日期、主诉、现病史、诊断、辨证等 21 个字段
- **处方药材提取**：支持 Grid 布局（药名/剂量分列）和内联文本（"甘草片20g太子参40g"）两种格式
- **OCR 医学纠错**：内置常见 OCR 误识别纠正表（如 "心季"→"心悸"、"神疲之力"→"神疲乏力"），自动修正医疗术语
- **字段防污染提取**：同线容差控制（±12px）、右侧面板边界过滤（x>1200 排除历史记录）、同线/多行去重、多行专用停止词，确保各字段内容互不串读
- **智能合并策略**：同一患者多张图片合并为一行，主视图值优先采用（布局完整），截断时自动回退到最长值
- **初诊/复诊自动判断**：根据右侧面板历史就诊记录自动识别就诊类型
- **非医疗图片过滤**：两级验证（结构标签 + 中文医疗内容检测），自动跳过架构图、流程图等非病历图片
- **去重写入**：基于姓名+日期+主诉去重，避免重复写入

### 快速开始

#### 环境要求

- Python 3.10+
- 依赖安装：

```bash
cd tcm_extractor
pip install -r requirements.txt
```

依赖清单：`rapidocr_onnxruntime>=1.4.0`、`openpyxl>=3.1.0`、`Pillow>=10.0.0`

#### 运行

```bash
# 处理单张图片
python main.py 1.1.jpg

# 批量处理整个目录
python main.py ./Case原始测试资料/

# 指定输出文件名
python main.py ./Case原始测试资料/ --output 6月记录.xlsx

# 持续监听目录（新增截图自动处理）
python main.py ./截图/ --watch
```

### 图片命名约定

同一患者的多张截图使用相同前缀，用 `.` 分隔序号：

```
1.1.jpg    ← 患者1 第1张截图（主视图）
1.2.jpg    ← 患者1 第2张截图（处方详情等）
2.1.jpg    ← 患者2 第1张截图
2.2.jpg    ← 患者2 第2张截图
```

同组图片会被合并为一行记录。合并时基本信息（姓名、电话等）取第一个非空值，病历文本字段优先采用主视图（第一张图片）的值，处方药材跨图片模糊去重。

生成 `.xlsx` Excel 文件，包含两个 Sheet：

**门诊记录**（主表）：每行一位患者的一次就诊记录，21 列

| 字段 | 说明 |

| 既往史 | 既往病史 |

### 配置

编辑 `config.py` 可调整以下参数：

```python
OUTPUT_EXCEL = "门诊记录.xlsx"   # 输出文件名
MAX_RETRIES = 3                  # OCR 失败重试次数
IMAGE_MAX_SIDE = 2000            # 图片最大边长（像素），None 不压缩
DEDUP_KEYS = ["姓名", "就诊日期时间", "主诉"]  # 去重主键
```

### 关键设计说明

#### 字段提取与防污染机制

字段提取是本工具最核心的部分。门诊系统截图的布局复杂——主内容区、侧边栏患者列表、右侧历史记录面板共存于同一画面，极易导致字段内容互相污染。系统通过以下多层机制解决：

**同线值收集（Strategy A）**：找到字段标签后，在 ±12px 容差内收集同行右侧的值文本块。同线值 ≥ 10 字符时直接返回，不进入多行收集，从根本上避免越界污染。同时排除右侧面板区域（x > 1200）的文本块。

**多行值收集（Strategy B）**：仅当同线值不足 10 字符时触发。从标签位置向下扫描，受以下停止条件保护：字段标签前缀匹配、左侧短文本标签检测、"确诊"/"否认"/"患者"等多行专用停止词、日期/电话等非标模式、y 间距超 40px 自动截断。

**同线/多行去重**：通过 `same_line_texts` 集合记录已收集的同线值，多行循环中跳过相同文本，避免同一内容被收集两遍。

#### 右侧面板过滤

门诊系统右侧约 x > 1200 区域显示患者历史就诊列表（疾病名+医师+日期），这些文本会严重干扰字段提取。系统在字段提取和诊断/辨证提取中统一应用 `RIGHT_PANEL_X_THRESHOLD = 1200` 过滤。

#### 智能合并策略

同一患者的多张图片按以下策略合并为一行：

- **基本信息**（姓名、电话、医师等）：取第一个非空值，通常来自主视图
- **病历文本**（主诉、现病史等）：首条记录（主视图）值 ≥ 10 字符时优先采用；否则取所有记录中最长的非空值。这确保主视图的干净值不被处方详情图的污染值覆盖
- **处方药材**：跨图片模糊去重，保留已知药名和合理剂量
- **就诊日期**：取最新日期

#### 侧边栏过滤

门诊系统截图左侧约 200px 为患者列表和导航栏，其文本会干扰病历字段提取。系统自动过滤 `x_center < 200` 的文本块。

#### 姓名提取策略

按优先级依次尝试三种策略：

1. 在主内容区域查找性别/年龄块左侧的中文姓名
2. 回退到头部区域 (y=280~350) 搜索
3. 从侧边栏患者列表提取当前选中项（无"预"后缀的条目），并裁剪 OCR 伪影（如 "孙政圖" → "孙政"）

#### 初诊/复诊判断

自动判断逻辑按以下优先级：

1. 右侧面板中出现历史就诊日期条目（MM-DD 格式，排除当前就诊日期）→ 复诊
2. 独立的"初诊"或"复诊"文本块
3. 无信号时留空，由人工补充

#### OCR 医学纠错

内置词级纠错表，对提取后的病历文本做自动修正。当前覆盖的纠错包括：心季→心悸、胸闵→胸闷、纳呆→纳差、神疲之力→神疲乏力。纠错仅作用于词级匹配（非单字），避免过度修正。可通过编辑 `extractor.py` 中的 `_MEDICAL_CHAR_FIX` 字典扩展。

#### 药材模糊去重

处理 OCR 对相似字形的误识别（如"黄苓片"/"黄芩片"/"黄答片"），通过字符混淆映射表自动合并，保留已知药名。

#### 非医疗图片过滤

两级验证确保只处理有效病历图片：第一级检查是否包含结构标签（主诉、现病史等），第二级验证提取到的字段值是否包含中文医疗术语（排除架构图等含代码/英文的误识别）。如果图片提取到了处方药材，则跳过第二级检查（纯处方图也有效）。

### 已知局限与后续优化方向

- **现病史截断**：当现病史内容跨越多行且超出单张截图可见范围时，提取值可能在图片边缘处被截断。可通过多张图片互补缓解
- **OCR 剂量误差**：RapidOCR 对部分数字识别不准（如 24g 误读为 66g），建议对关键处方进行人工复核
- **性别/年龄缺失**：部分截图（如处方详情页）不含患者头部信息，需依赖同组其他图片补充
- **就诊类型检测**：依赖右侧面板历史日期条目，对于面板被折叠或不可见的截图可能无法判断
- **药材名 OCR 变体**：已覆盖常见混淆字符（苓/芩/答等），但罕见变体需持续补充
- **OCR 纠错表扩展**：当前纠错表仅覆盖最常见的误识别，可通过观察提取结果持续补充 `_MEDICAL_CHAR_FIX`
